## [🦄文章&教程](https://xiaobot.net/p/python_weekly)

1、[英伟达终于为 CUDA 添加了原生 Python 支持](https://thenewstack.io/nvidia-finally-adds-native-python-support-to-cuda/)

NVIDIA 在 CUDA 工具包中实现原生支持 Python，将 Python 提升为一等公民，开发者不需要了解 C/C++，可以更方便使用 CUDA 工具包了。

2、[Django 5.2 版本有什么新特性？](https://adamj.eu/tech/2025/04/07/django-whats-new-5.2/)

Django 5.2 版本已发布了，文章详细介绍了多项新功能和改进：自动模型导入、复合主键支持、表单自定义、内容协商、模板块标签、URL 生成改进、测试优化等。

3、[掌控个人数据：自托管日历的解决方案](https://emilygorcenski.com/post/owning-my-own-data-part-1-integrating-a-self-hosted-calendar-solution/)

作者希望摆脱对 Google Calendar 等服务的依赖，通过自托管方案掌握个人日历数据，介绍了从 CalDAV 到 iCalendar 的探索过程，利用 Python 实现日历数据的格式转换。

4、[用 PySide6 和 SQLAlchemy 开发桌面便签应用](https://www.pythonguis.com/examples/pyside6-desktop-sticky-notes)

这篇教程详细介绍了如何开发一款桌面便利贴应用，使用 PySide6 和 SQLAlchemy，实现无边框可拖动便签、多便签管理、自定义便签样式、集成系统托盘等功能。

5、[PEP-784：将 Zstandard 添加到标准库](https://peps.python.org/pep-0784/)

一则草稿态 PEP，提议添加一个新的标准库，将 Meta 的 zstd 库作为默认实现。Zstandard 提供高压缩比和快速解压，优于 zlib 和 bzip2。

6、[在 Django Admin 中用 Celery 运行后台任务](https://testdriven.io/blog/django-admin-celery)

如何在 Django 项目中处理长时间运行的任务？如何将 Celery 与 Django 集成？如何在 Django 管理后台触发任务，并获取实时的任务状态？

7、[用于高性能终端应用的算法](https://textual.textualize.io/blog/2024/12/12/algorithms-for-high-performance-terminal-apps/)

Textual 项目的作者分享了如何在终端中实现高性能图形用户界面的技术细节，包括如何解决多窗口重叠问题、如何快速确定哪些窗口在可见区域、如何优化终端应用的渲染速度，等等。

8、[使用 JAX 重新实现 word2vec](https://eli.thegreenplace.net/2025/reproducing-word2vec-with-jax)

word2vec 是一种用于生成词嵌入（word embeddings）的模型，文章详细介绍了如何用 JAX 重新实现它。

9、[在 LLM 时代提升推荐系统与搜索的效果](https://eugeneyan.com/writing/recsys-llm/)

推荐系统和搜索技术在如今的 LLM 时代，发生了哪些演变？文章讨论了业界搜索和推荐系统在过去一年多的发展，涵盖模型架构、数据生成、训练范式和统一框架等方面。

10、[剖析代码机器人：LLM 是如何理解 null 值的？](https://dmodel.ai/nullability-gentle)

大模型能否正确地理解 null 值，这对生成代码至关重要。文章介绍了如何通过模型输出和内部状态来评估模型对空值的理解，展示了模型训练及代码上下文对大模型的影响。

11、[Python 脚本为何频遭封禁？小心别中招！](https://dev.to/snappytuts/pythons-most-banned-scripts-getting-you-blacklisted-55n4)

为什么有些 Python 库会被云服务供应商屏蔽？为什么 Scrapy、BeautifulSoup、Selenium 等库被识别为高风险库？机器人开发者如何应对这些限制？

12、[“椭圆的” Python 编程](https://susam.net/elliptical-python-programming.html)

`--(...==...)` 等于 1，`--(...==...)--(...==...)` 等于 2，利用这点可以写出一长串内容来表示“hello, world”，但这种晦涩难懂的代码，是对过度追求技巧而忽视可读性的讽刺。

## [🐿️项目&资源](https://xiaobot.net/p/python_weekly)

1、[GamingAgent：个人电脑上的游戏智能体](https://github.com/lmgame-org/GamingAgent)

旨在提供一个简易部署计算机运行智能体（CUA）的解决方案，可在 PC 和个人笔记本上运行。

![](https://img.pythoncat.top/2025-04-11-computer-game.png)

2、[memo：管理你的 Apple Notes 和 Reminders](https://github.com/antoniorodr/memo)

Python 开发的简单命令行工具，用于管理 Apple Notes 和 Apple Reminders，从终端创建、搜索和组织你的笔记。

3、[Nugget：释放 iOS 设备的潜能](https://github.com/leminlimez/Nugget)

一个针对 iOS 17.0 至 18.1.1 设备的工具，通过漏洞解锁设备的隐藏功能（启用动态岛、设备伪装、动态壁纸与 AOD等），禁用不必要的系统服务。（star 2.5K）

4、[no-code-architects-toolkit：将常用 API 整合成免费的 API](https://github.com/stephengpope/no-code-architects-toolkit)

支持音频转录、多语言翻译、为视频添加字幕，并能执行复杂的媒体处理，适用于内容创作。它还支持跨云服务的文件管理，包括 Google Drive、Amazon S3、Google Cloud Storage 和 Dropbox。

5、[curl-impersonate：特别版 curl，可模拟 Chrome 和 Firefox 等浏览器](https://github.com/lwthiker/curl-impersonate)

一款特殊的 curl 构建，可模拟四大浏览器：Chrome、Edge、Safari 和 Firefox。能够执行与真实浏览器完全相同的 TLS 和 HTTP 握手。（star 5.1K）

6、[Wikipedia-API：维基百科 API 的 Python 封装 ](https://github.com/martin-majlis/Wikipedia-API)

便于用 Python 检索维基百科的信息，支持从维基百科提取文本、章节、链接、分类、翻译等。

7、[adk-python：用于开发、评估和部署复杂 AI agent](https://github.com/google/adk-python)

谷歌开源的 Agent Development Kit (ADK)，助力打造高效、智能的 AI agent，特色：代码优先、多代理架构、丰富的生态、灵活编排、原生流支持、状态/记忆与组件管理。（star 3.2K）

8、[smartfunc：将文档字符串转成 LLM 函数](https://github.com/koaning/smartfunc)

通过装饰器将文档字符串直接转换为 LLM 函数，支持 Jinja2 模板，支持异步函数，使用 Pydantic 模型定义响应结构，确保输出格式一致。

9、[textcase：文本拼写格式转换库](https://github.com/zobweyt/textcase)

支持多种拼写格式的转换（如 snake_case、kebab-case、camelCase 等），智能识别缩写（如 HTTPRequest）并支持非 ASCII 字符。

10、[BabelDOC：PDF 文档翻译器](https://github.com/funstory-ai/BabelDOC)

PDF 科学论文翻译和双语对照库。有在线版沉浸式翻译，也支持自部署。（star 1.6K）

![](https://img.pythoncat.top/2025-04-11-babeldoc.png)

11、[arborparser：将有层次的结构化文本解析成树形表示](https://github.com/Jimmy228676/arborparser/tree/main)

解析结构化文本，并将它们转换为基于层级标题的树形式，适用于处理具有嵌套标题的文档，如大纲、报告和技术文档。（分享自@Enoch）

12、[DeepSeek-V3：DeepSeek 开源大模型](https://github.com/deepseek-ai/DeepSeek-V3)

强大的混合专家（MoE）大模型，总参数量为 671B，用了多头潜在注意力（MLA）和 DeepSeekMoE 架构，开创了无辅助损失的负载均衡策略。（star 95.4K）
