## [🦄文章&教程](https://xiaobot.net/p/python_weekly)

1、[Python 3.14 新特性盘点，更新了些什么？](https://pythoncat.top/posts/2025-05-08-python-3.14)

Python 3.14 已进入特性冻结期，更新的内容很多，有哪些值得特别关注的呢？文章全面梳理了此版本的变更：类型提示的惰性求值、全新的尾调用解释器、实验性的 JIT 编译器、无侵入调试接口、REPL 的语法高亮、模板字符串和 Zstandard 压缩，等等。

2、[Python π（3.14）](https://iscinumpy.dev/post/python-314/)

也是一篇 Python 3.14 的主要功能盘点，相比上一则分享的内容，它还介绍了比较多细小的特性变更，可以对照阅读。另外，值得注意的是，博主还发布过 Python 3.7 以来所有主版本的新特性盘点。

3、[再见 PyCon APAC，开启 PAO 新篇章](https://iqbalabdullah.net/en/posts/2025/03/the-final-pycon-apac/)

回顾了 PyCon APAC（PyCon 亚太组织） 自 2010 年在新加坡创办以来 15 年的发展历程，现社区更名为 Python Asia Organization (PAO) ，文章解释了原因和相关规划。（附：PyCon JP 成员评论，[PAO 对日本的 Python 社区意味着什么？](https://www.bilibili.com/video/BV1P6VzziEbE)）（分享自@NoHeartPen）

4、[让 AI 做语义单元测试](https://www.alexmolas.com/2025/04/09/semantic-unit-testing.html)

一种创新的测试方法：让大模型来分析代码是否符合文档描述。作者开发的 suite 库（Semantic UnIt TEsting）通过分析函数代码和文档注释，无需实际运行就能发现潜在问题。

5、[解密 Asyncio：从零开始手搓一个](https://dev.indooroutdoor.io/asyncio-demystified-rebuilding-it-from-scratch-one-yield-at-a-time)

深入浅出讲解了 Python 异步编程的底层原理，一步步手搓一个简化版的 Asyncio。通过极简的代码还原了 Asyncio 的核心机制，帮助读者彻底理解 await 背后的“魔法”。

6、[Python 依赖管理的新进展：PEP 751 锁文件标准](https://reinforcedknowledge.com/python-project-management-and-packaging-pep-751-update-and-some-of-the-remaining-issues-of-packaging/)

介绍了 Python 依赖管理的最新进展，特别是 PEP 751 标准的锁文件格式。文章详细解释了锁文件的必要性，pylock.toml 的特点及其解决的问题，并探讨了 Python 打包领域仍存在的一些挑战，如原生依赖和加速器依赖包等问题。

7、[用温度参数调整随机权重](https://www.boristhebrave.com/2025/05/07/fiddling-weights-with-temperature/)

介绍了 AI 领域中简单而好用的权重调整技术：用温度参数来控制随机选择。温度为 1 时保持原始权重，小于 1 时偏好高权重项，大于 1 时趋向均匀分布。

8、[使用 MCP 和 Amazon Q 开发新闻智能体](https://eugeneyan.com//writing/news-agents/)

作者基于 MCP 和 Amazon Q CLI 开发了一个新闻摘要智能体，支持并行处理和自动摘要。使用 tmux 管理多个进程，每个子智能体负责一部分新闻源，最后由主智能体整合生成每日新闻。

9、[Python 任务队列的扩展指南](https://judoscale.com/blog/scaling-python-task-queues)

探讨了 Python 任务队列的扩展性问题，对比了 Redis 和 RabbitMQ 两种消息中间件的特点。还介绍了垂直扩展（增加单个 worker 的资源）和水平扩展（增加 worker 数量）两种策略，以及扩展 Celery 和 RQ 的最佳实践。

10、[pip 25.1 发布，支持依赖组管理](https://ichard26.github.io/blog/2025/04/whats-new-in-pip-25.1/)

pip 25.1 带来了多项重要更新：支持 PEP 735 的依赖组管理；新增了包安装进度条；支持断点续传下载；还支持生成 PEP 751 标准的锁文件。另外从版本开始，不再支持 Python 3.8 了。

11、[Django 模板语言的小怪癖](https://blog.lilyf.org/posts/django-template-quirks/)

作者正在用 Rust 重写 Django 模板引擎，发现了几个有趣的边界情况：科学计数法不支持负指数、upper 过滤器对 HTML 实体不安全、if 标签中的异常处理会导致反直觉的结果。

12、[Python 代码质量工具大全](https://techbeamers.com/python-code-quality-tools/)

介绍了一套 Python 代码质量工具链：静态分析（Pylint、Flake8、Bandit）、代码格式化（Black、isort）、类型检查（Mypy、Pytype）、安全扫描（Safety、Snyk）、测试覆盖（Coverage\.py）、性能分析（cProfile）等。

## [🐿️项目&资源](https://xiaobot.net/p/python_weekly)

1、[ty：Rust 开发的 Python 类型检查工具](https://github.com/astral-sh/ty)

Ruff 和 uv 所属的 Astral 团队出品，终于将 Rust 风吹到了静态类型检查工具上！目前仅为预发布，仍在积极开发中。（star 4.2K）

2、[WeClone：用微信聊天记录创造数字分身 ](https://github.com/xming521/WeClone)

使用微信聊天记录微调大语言模型，让大模型有“那味儿”，并绑定到聊天机器人，实现自己的数字分身。（star 3.3K）

3、[AI-Media2Doc：将音/视频转化为小红书/公众号/知识笔记/思维导图](https://github.com/hanshuaikang/AI-Media2Doc)

基于 AI 的音视频到图文创作助手，使用 Vue 开发的 Web 前端，本地部署，无需登录注册。支持小红书/公众号/知识笔记/思维导图/内容总结等多种文档风格。

![AI生成思维导图](https://img.pythoncat.top/2025-05-09-media2doc.png)

4、[blast：AI 浏览 Web 的高性能服务引擎](https://github.com/stanford-mast/blast)

为你的应用添加 AI 浏览 Web 功能，兼容 OpenAI 的 API 提供并发和流式处理，自动缓存和并行化降低成本。

5、[contextgem：LLM 提取文档的结构化数据](https://github.com/shcherbak-ai/contextgem)

以少量代码从文本和图像中提取结构化数据和洞察，支持神经分割、并发 IO 处理、嵌套上下文提取、多语言支持等。

6、[Fluent-M3U8：PySide6 开发的跨平台 m3u8/mpd 下载器](https://github.com/zhiyiYo/Fluent-M3U8)

基于 PySide6 和 QFluentWidgets 开发，GUI 简洁易用，兼容 M3U8 和 MPD 视频、多线程 M3U8 下载、下载任务管理。（star 1K）

7、[Python-UIAutomation-for-Windows: Windows 平台的 Python UI 自动化库](https://github.com/yinkaisheng/Python-UIAutomation-for-Windows)

封装了微软的 UIAutomation 接口，用 Python 脚本自动化操作各种 Windows 桌面应用，包括 MFC、Windows Form、WPF、现代 UI（Metro UI）、Qt、IE、Firefox、Chrome 以及基于 Electron 的应用。（star 2.9K）

8、[video-subtitle-extractor：提取视频硬字幕，生成srt文件](https://github.com/YaoFANGUK/video-subtitle-extractor)

本地化的视频硬字幕提取工具，自动检测字幕区域并识别字幕内容，无需依赖第三方在线 OCR 服务。支持去除水印、台标、重复字幕等功能，支持 GPU 加速。（star 7.2K）

9、[epub-utils：EPUB 文件解析与元数据提取](https://github.com/ernestofgonzalez/epub-utils)

Python 编写的 EPUB 处理工具，在终端中解析和检查 EPUB 电子书。支持解析和校验标题、作者、标识符等元数据信息，并能高亮显示 XML 内容。

10、[novelWriter：纯文本小说写作编辑器](https://github.com/vkbo/novelWriter)

Python 开发的纯文本编辑器，将长篇小说拆分为多个小文档进行组织和管理，采用类 Markdown 的极简标记语法，并扩展了元数据注释、梗概和交叉引用功能。支持多平台（Linux、Windows、MacOS）。（star 2.4K）

![](https://img.pythoncat.top/2025-05-09-novel-writer.png)

11、[asciimatics：跨平台文本界面与 ASCII 动画开发库](https://github.com/peterbrittain/asciimatics)

开发全屏文本界面和 ASCII 动画，支持彩色文本、光标控制、键盘和鼠标输入、窗口自适应、屏幕抓取，还内置了丰富的动画特效、图像转 ASCII、表单控件等高级功能，适合做命令行工具、交互式表单、复古风动画等。（star 4K）

12、[谷歌的提示词工程白皮书](https://drive.google.com/file/d/1AbaBYbEa_EbPelsT40-vj64L-2IwUJHy/view)

一份在线 PDF 文档《Prompt Engineering》，出自 Google 的提示词工程白皮书。
