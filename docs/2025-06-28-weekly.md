## [🦄文章&教程](https://weekly.pythoncat.top)

1、[我第一个用 AI 生成的开源库](https://lucumr.pocoo.org/2025/6/21/my-first-ai-library/)

Flask 作者 Armin Ronacher 分享了用 AI 开发 XML 解析库的经历。他让 Claude 从零开始写了一个容错性强的 XML 解析器，用于处理 LLM 生成的不规范 XML 内容，1100 行解析器代码和 1000 行测试代码，还配置了完整的 CI 流程、PyPI 发布，整个过程仅用时 30-45 分钟。

2、[uv 和 PEP 723 的趣味探索](https://www.cottongeeks.com/articles/2025-06-24-fun-with-uv-and-pep-723)

介绍了如何用 uv 和 PEP 723 内联脚本元数据来创建自包含的 Python 脚本。通过在脚本中嵌入依赖信息和 Python 版本信息，uv 能自动创建虚拟环境并安装依赖，让 Python 脚本像 Go 程序一样可以直接执行。

3、[AI 会取代初级开发者吗？Guido 和其他人怎么看](https://blog.adarshd.dev/posts/pycon-us-ai-and-future-of-programming/)

作者在 PyCon US 上采访了 Python 社区的大 V，包括 Guido van Rossum、Anthony Sottile 等，他们普遍认为 AI 不会取代开发者，但会改变工作方式，还强调理解代码比依赖 AI 生成更重要，AI 更适合处理样板代码和重复性任务。

4、[修复 Python 属性](https://willmcgugan.github.io/fixing-python-properties/?featured_on=pythonbytes)

作者分享了如何用描述符替代 @property 装饰器，解决属性 setter 接受灵活类型时的类型检查报错问题，从而能更自然地设置属性值。

5、[Python 现在可以运行 Mojo 了](https://koaning.io/posts/giving-mojo-a-spin/)

介绍了 Python 调用 Mojo 代码的新功能，在质数计算测试中，Mojo 版本比纯 Python 快 40 倍，比 NumPy 快 23 倍，展现了 Mojo 的性能优势。

6、[Django 快照测试新工具](https://adamj.eu/tech/2025/06/24/django-introducing-inline-snapshot-django/)

作者开源了 inline-snapshot-django 包，专门用于 Django 项目的 SQL 查询快照测试。该工具通过上下文管理器自动捕获和比较 SQL 查询指纹，相比 Django 原生的方法提供更详细的调试信息和自动化的快照管理功能。

7、[用原生连接池将 Django 数据库延迟降低 50-70ms](https://saurabh-kumar.com/articles/2025/06/cut-django-database-latency-by-50-70ms-with-native-connection-pooling/)

Django 5.1 引入了原生数据库连接池功能，通过复用 PostgreSQL 连接，替代各种复杂的创建新连接的方法，将数据库连接开销降低 60-80%，响应时间提升 10-30%。文章介绍了配置步骤和连接池参数，以及如何监控性能提升。

8、[等车时间悖论：为什么我的公交车总是迟到？](https://jakevdp.github.io/blog/2018/09/13/waiting-time-paradox/)

作者通过 Python 模拟和概率论分析揭示了等车时间悖论：当公交车平均每 10 分钟一班时，乘客的平均等车时间竟然也是 10 分钟而非 5 分钟。文章用泊松过程和指数分布解释了这一现象，并用西雅图真实公交数据来验证理论预测，但发现日常现象跟理论预测还是存在差异。

9、[Python 中的卡尔曼滤波器和贝叶斯滤波器](https://github.com/rlabbe/Kalman-and-Bayesian-Filters-in-Python/blob/master/README.md)

一份用 Jupyter Notebook 写的交互式教程，专门介绍卡尔曼滤波器和贝叶斯滤波器的原理与实现。两个算法在航空航天、机器人、金融等领域有广泛应用，作者从传感器噪声问题出发，通过 Python 代码和可视化演示，深入浅出地做了入门科普。

10、[从 virtualenvwrapper 切换到 direnv、Starship 和 uv](https://treyhunner.com/2024/10/switching-from-virtualenvwrapper-to-direnv-starship-and-uv/)

作者分享了从 virtualenvwrapper 迁移到现代 Python 环境管理工具的经历。通过配置 direnv 实现项目目录自动激活虚拟环境，使用 venv 和 workon 命令简化虚拟环境创建和项目切换流程，使用 Starship 定制简洁的命令行提示符。

11、[如何思考 Agent 框架？](https://blog.langchain.com/how-to-think-about-agent-frameworks/)

LangChain 团队深度分析了 Agent 框架的设计思路，指出构建可靠 Agent 系统的核心挑战是确保 LLM 在每个步骤都有适当的上下文。文章区分了 Workflow（预定义代码路径）和 Agent（LLM 动态控制）两种模式，强调大多数生产环境的 Agent 系统实际是两者的结合，并对比了 100 多个 Agent 框架的设计理念。

12、[9 年来棘手 Bug 的经验教训](https://henrikwarne.com/2025/06/15/lessons-from-9-more-years-of-tricky-bugs/)

作者总结了过去 9 年里学到的经验教训，从编码、测试、调试三个维度分析了 12 个关键问题，包括空值处理、日期逻辑、数据格式升级、修改后忘推送等，强调了良好日志记录、与同事讨论、用最小案例复现的重要性。

## [🐿️项目&资源](https://weekly.pythoncat.top)

1、[gemini-cli：终端中的 Gemini AI 智能体](https://github.com/google-gemini/gemini-cli)

Google 开源的命令行 AI 工作流工具，可以查询和编辑大型代码库，支持超过 100 万 token 的上下文窗口，能够从 PDF 或草图生成新应用，自动化操作任务如查询 PR 或处理复杂的代码合并，还可以通过 MCP 服务器连接各种工具和媒体生成能力。（star 33.8K）

2、[nxtscape：开源的 AI 智能体浏览器](https://github.com/nxtscape/nxtscape)

开源的 AI 智能体浏览器，隐私优先，是 Arc、Dia 和 Perplexity Comet 替代方案。可使用自己的 API 密钥或通过 Ollama 运行本地模型。（star 1K）

3、[augmentoolkit：创建定制化的 LLM](https://github.com/e-p-armstrong/augmentoolkit)

专为创建领域专家 AI 而设计的数据生成工具，可以上传文档并一键生成完全训练好的定制 LLM。支持离线运行，无需外部 API 密钥，能够让 AI 学习特定领域知识，成为领域专家。（star 1.6K）

4、[BitNet：1-bit 大模型推理框架](https://github.com/microsoft/BitNet)

微软开源的 1-bit 大模型官方推理框架，支持 BitNet b1.58 等模型的快速无损推理。在 ARM CPU 上实现 1.37x 到 5.07x 的加速，能耗降低 55.4% 到 70.0%，在 x86 CPU 上加速 2.37x 到 6.17x，能耗降低 71.9% 到 82.2%。可在单个 CPU 上运行 100B 参数的 BitNet 模型，达到人类阅读速度。（star 20.3K）

![](https://img.pythoncat.top/2025-06-27-bitnet.jpg)

5、[local-deep-research：本地深度研究助手](https://github.com/LearningCircuit/local-deep-research)

AI 驱动的研究助手，将复杂问题转化为全面的引用报告，跨越学术数据库、科学仓库、网络内容和私人文档集合等多样化知识源。支持隐私优先、灵活配置、全面分析和透明引用，提供多种研究模式和高级功能。（star 3K）

6、[pyfuze：Python 项目打包工具](https://github.com/TanixLu/pyfuze)

将 Python 项目打包成可执行文件的工具，基于 cosmopolitan 和 uv 构建。提供三种打包模式：Bundle 模式包含 Python 和所有依赖，Online 模式生成小巧的跨平台包，Portable 模式创建独立的跨平台可执行文件。支持 macOS、Linux 和 Windows 平台。

7、[nbdev：基于 Jupyter Notebook 的开发平台](https://github.com/AnswerDotAI/nbdev)

通过轻量级标记语言编写 notebook，自动生成高质量文档、测试、持续集成和打包。支持文档自动生成并托管在 GitHub Pages，发布到 PyPI 和 conda，笔记本与源代码双向同步，并提供 Git 友好的笔记本处理功能。（star 5.1K）

8、[fastapi_mcp：将 FastAPI 项目暴露为 MCP 工具](https://github.com/tadata-org/fastapi_mcp)

将 FastAPI 端点暴露为 MCP 工具的库，内置身份验证。支持零配置或最小配置，支持灵活部署，使用 ASGI 传输直接与 FastAPI 应用通信，提供原生依赖注入和统一基础设施。（star 5.6K）

9、[Awesome-ML-SYS-Tutorial：机器学习系统学习笔记](https://github.com/zhaochenyang20/Awesome-ML-SYS-Tutorial)

一份机器学习的笔记和代码合集，涵盖 RLHF 系统开发、SGLang 学习、调度路由、分布式训练等核心主题。包含深入的源码解析、实践经验分享和开发指南，是学习 ML + SYS 领域的优质资源。（star 2.7K）

10、[machine-learning-visualized：机器学习算法可视化教程](https://github.com/gavinkhung/machine-learning-visualized)

通过 Jupyter Notebook 从第一性原理实现和推导机器学习算法的可视化教程。每个算法都有完整的数学推导和训练过程可视化，涵盖神经网络、逻辑回归、感知机、主成分分析、K-means 聚类和梯度下降等核心算法，还提供了基于 Marimo 的交互式笔记本来观察权重对损失函数的影响。

![](https://img.pythoncat.top/2025-06-27_neural_network.png)

11、[code-graph-rag：多语言代码库图谱 RAG 系统](https://github.com/vitali87/code-graph-rag)

基于图谱的多语言代码库检索增强生成系统，使用 Tree-sitter 解析代码并构建知识图谱。可以用自然语言查询代码库结构和关系。

12、[FeedMe：AI 驱动的 RSS 阅读器](https://github.com/Seanium/FeedMe)

用 AI 重新定义 RSS 阅读体验的轻量级阅读器，支持多源 RSS 聚合、AI 摘要生成、定时更新机制、分类浏览和主题切换。可轻松部署到 GitHub Pages 或使用 Docker 本地部署。
