## [🦄 Articles & Tutorials](https://xiaobot.net/p/python_weekly)

1、[Why 'if not list' is 2x Faster Than len() in Python](https://blog.codingconfessions.com/p/python-performance-why-if-not-list)

When checking if a Python list is empty, `if not mylist` is more elegant and faster than `if len(mylist) == 0`. The article explains why the former is faster by analyzing CPython's virtual machine instructions and object memory access patterns. (Note: This article is from a paid newsletter, with the first half freely accessible)

2、[Cracking the Python Monorepo](https://gafni.dev/blog/cracking-the-python-monorepo/)

Introduces methods for building efficient Python monorepos using `uv` and `Dagger`. `uv` manages dependencies and workspaces, while `Dagger` defines containerized build processes, achieving efficient modular builds and cache reuse.

3、[Performance of the Python 3.14 tail-call interpreter](https://blog.nelhage.com/post/cpython-tail-call/)

Recently, Python 3.14's tail-call interpreter has been eye-catching (10-15% performance improvement). However, after analysis, the author found that these improvements mainly come from accidentally bypassing a regression issue in LLVM 19. Under a more reasonable baseline, the performance improvement is only 1-5%.

4、[Python interpreter adds tail calls](https://lwn.net/Articles/1010905/)

Before introducing tail calls, CPython had three different interpreters: a switch-based bytecode interpreter, a computed goto-based bytecode interpreter, and a micro-operation interpreter. The article introduces the problems with previous interpreters and the optimization implementation of the new interpreter.

5、[Beating LinkedIn "Queens" with Python](https://mathspp.com/blog/beating-linkedin-queens-with-python)

LinkedIn has been publishing daily "Queens" puzzles for about a year. The author shares their Python program that can solve all Queens puzzles published by LinkedIn.

6、[Please Stop Asking About Singleton](https://frostming.com/2025/singleton/)

The author believes: In Python, you don't need Singleton; if you do, just use module-level variables. The article points out problems with several popular singleton implementation methods and finally provides a "simple and straightforward" solution.

7、[Creating static map images with OpenStreetMap, Web Mercator, and Pillow](https://alexwlchan.net/2025/static-maps)

Introduces a method for creating static maps without relying on complex visualization libraries: downloading maps from OpenStreetMap, converting latitude/longitude coordinates to pixel coordinates, and finally drawing colored markers on the map.

8、[PEP 779 – Criteria for supported status for free-threaded Python](https://peps.python.org/pep-0779/)

PEP-703 describes three development stages for free-threaded Python. This PEP establishes clear expectations and requirements for transitioning to the second stage, including evaluation criteria for performance, stability, maintainability, and memory usage.

9、[Alternatives to Python's "break" statement](https://www.pythonmorsels.com/break-statement-alternatives/)

Python for loops can use the break statement to exit loops. The author suggests we can use other alternative implementations based on different purposes, including using the in operator, any() function, next(), and itertools.takewhile().

10、[My Python commits: February 2025](https://vstinner.github.io/python-commits-february-2025.html)

Python core developer Victor Stinner had 18 commits merged in February. The article summarizes and records these code contributions.

11、[An oral history of Bank Python](https://calpaterson.com/bank-python.html)

What Python software are many major investment banks using? The article introduces several core subsystems of "Bank Python": Barbara object key-value store, Dagger dependency management, Walpole bank-grade job scheduling, and MnTable proprietary table processing.

12、[Intricate interleaved iteration](https://nedbatchelder.com/blog/202501/intricate_interleaved_iteration.html)

The author wants to answer this question: If you need to store all values, why still use generator syntax? Through implementing interleaved arrangement of blog sidebar tags and years, the article introduces flexible use of generators and trade-offs between different implementation approaches.
