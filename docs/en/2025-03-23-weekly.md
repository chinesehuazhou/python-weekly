This week we're featuring 11 articles, 12 open source projects.

## [🦄Articles & Tutorials](https://weekly.pythoncat.top)

1、[How to solve FastAPI's large file transfer problem?](https://so1n.me/2024/10/01/how_to_solve_fastapi_lager_file/)

Why is FastAPI/Starlette slower than other frameworks when handling large file transfers? In file upload and download scenarios, the author compares the performance differences of various implementation approaches, analyzes the root causes, and shares multiple performance optimization solutions.

![](https://img.pythoncat.top/2025-03-22-diff-large-file-download.png)

2、[Operationalizing Python – Part 1: Why Does It Hurt?](https://www.leejustin.com/Operationalizing-Python-Part-1-Why-Does-It-Hurt-1b8703f0ba8f80e4a9f4d9527f200aeb)

Python is great to write, but can be frustrating to maintain. The article analyzes some reasons why Python can feel painful, such as clashing programming cultures and package dependency management, and finally shares suggestions for improving maintainability.

3、[Python Discord 2024 Survey Report](https://pydis.wtf/~joe/survey-results-2024.html)

The 2024 survey results from the Python Discord community, covering user programming experience, community engagement, learning resource usage, help seeking and providing, community activity experience, voice channel usage, and open source contribution intentions.

4、[A Very (!) Early Play With Astral's Red Knot Static Type Checker](https://jurasofish.github.io/a-very-early-play-with-astrals-red-knot-static-type-checker.html)

The team behind Ruff and uv is developing a Python type checker aimed at becoming an alternative to mypy. The article introduces Red Knot's installation, configuration, and usage methods, and demonstrates its significant speed advantages through comparisons with mypy and performance testing.

5、[Best Practises for A Performant Django Admin](https://hodovi.cc/blog/best-practises-for-a-performant-django-admin/)

Addressing Django Admin's performance issues when handling large datasets, the article introduces various optimization strategies: limiting sortable fields, adjusting pagination size, using estimated counts, caching, optimizing search fields, etc.

6、[PEP 776 – Emscripten Support](https://peps.python.org/pep-0776/)

PEP-776 proposes adding Emscripten as Tier 3 support in Python 3.14. Emscripten is a compiler toolchain that can compile C/C++ code to WebAssembly/JavaScript, suitable for browsers and JavaScript runtimes like Node.js.

7、[Binary search as a bidirectional generator](https://mathspp.com/blog/binary-search-as-a-bidirectional-generator)

The article proposes a binary search implementation based on bidirectional generators, using Python generator's send method to dynamically adjust algorithm behavior. This method can be extended to unbounded search, providing new implementation ideas.

8、[The Boolean Trap](https://read.engineerscodex.com/p/the-boolean-trap)

The author points out that using boolean parameters in functions or APIs can easily cause problems, such as poor code readability, limited extensibility, and potential errors, suggesting using enums as an alternative.

9、[Reinventing notebooks as reusable Python programs](https://marimo.io/blog/python-not-json)

Marimo is a new type of notebook that, unlike traditional Jupyter notebooks, saves as Python files, solving pain points like version control, code reuse, and testing, while supporting import and script execution capabilities.

10、[Python Programming and the Art of Knitting](https://compucademy.net/python-programming-and-the-art-of-knitting/)

What relationship could coding with code and knitting with yarn possibly have? Besides requiring patience, logic, and creativity, the article shares several interesting parallels.

11、[The "Active Enum" Pattern](https://blog.glyph.im/2025/01/active-enum.html)

The article points out that traditional enums have problems like maintenance difficulties and separation of behavior from data, thus proposing the "Active Enum" pattern, making enums not just data containers but also actively executing related operations. It demonstrates specific code examples, revealing its advantages, potential problems, and corresponding solutions.

## [🐿️Projects & Resources](https://weekly.pythoncat.top)

1、[docs: A collaborative note taking, wiki and documentation platform that scales](https://github.com/suitenumerique/docs)

A collaborative document editor developed with Django and React, serving as an open source alternative to Notion, Outline, and Confluence. (8.9K stars)

![](https://img.pythoncat.top/2025-03-22-docs.png)

2、[mcp-tool-kit: Agentic abstraction layer for building high precision vertical AI agents written in python](https://github.com/getfounded/mcp-tool-kit/tree/main)

MCP (Model Context Protocol) middleware for developing high-precision vertical AI agents.

3、[eventure: A powerful event-driven framework for simulations, games, and complex systems](https://github.com/enricostara/eventure)

A pure Python event-driven framework with features: event-driven, event sourcing, tick-based system, advanced event querying, out-of-the-box examples, type safety.

4、[html5lib-python: Standards-compliant library for parsing and serializing HTML documents and fragments in Python](https://github.com/html5lib/html5lib-python)

A pure Python HTML parsing library compliant with WHATWG HTML specifications. (1.2K stars)

5、[Run Python Online: Run Python code in the browser](https://cliprun.com/)

A website for executing Python scripts online, providing a Python compiler and IDE in the browser, with built-in common Python libraries and support for file upload/download.

6、[rusty-graph: High-performance graph database library developed in Rust](https://github.com/kkollsga/rusty-graph/tree/main)

Efficiently converts SQL relational data into knowledge graphs, implementing high-performance computing through Rust at the bottom layer, providing Python bindings to lower the usage barrier.

7、[aiopandas: Lightweight Pandas monkey-patch that adds async support to map, apply, applymap, aggregate, and transform](https://github.com/telekinesis-inc/aiopandas/tree/main)

Uses monkey patching to add async support to Pandas' map, apply, applymap, aggregate, and transform methods.

8、[basic-memory: Build persistent semantic graphs from AI assistant conversations](https://github.com/basicmachines-co/basic-memory)

A knowledge management system that uses conversations with large models to build persistent knowledge, saved as Markdown files.

9、[Python Developer Tooling Handbook](https://pydevtools.com/handbook/)

An online e-book aimed at helping you understand the complex ecosystem of developing Python packages: What exactly are uv, Poetry, Flit, Setuptools, and Hatch? What are their pros and cons? How to compare and choose? Additionally, it covers content on code review, formatting, and dependency management.

10、[mcp-agent: Build effective agents using Model Context Protocol and simple workflow patterns](https://github.com/lastmile-ai/mcp-agent)

A simple, composable agent development framework using MCP protocol and all patterns described in Building Effective Agents. (1.6K stars)

11、[InkyPi: E-Ink Display with a Raspberry Pi and a Web Interface to customize and update the display with various plugins](https://github.com/fatihak/InkyPi)

Open source, customizable E-Ink display providing a web interface usable on any networked device, easy to install and configure, supports timed playlists. (1K stars)

12、[physicsnemo: Open-source deep-learning framework for building, training, and fine-tuning deep learning models using state-of-the-art Physics-ML methods](https://github.com/NVIDIA/physicsnemo)

Builds, trains, and fine-tunes deep learning models using state-of-the-art SciML methods. Provides tools and optimization pipelines to develop AI models that combine physical knowledge with data for real-time predictions. (1.3K stars)
