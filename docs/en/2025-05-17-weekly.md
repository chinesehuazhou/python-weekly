## [🦄 Articles & Tutorials](https://xiaobot.net/p/python_weekly)

1、[The first year of free-threaded Python](https://labs.quansight.org/blog/free-threaded-one-year-recap) 

The Quansight team shares their achievements from the past year collaborating with Meta to advance GIL-free Python, including implementing support in numerous key packages (like NumPy, SciPy, etc.) and project tools, bringing multi-threading performance improvements to CPython 3.14.

2、[An Empirical Study on the Performance and Energy Usage of Compiled Python Code](https://arxiv.org/pdf/2505.02346)

An academic paper studying the performance and energy consumption of eight Python compilers (like PyPy, Numba, Codon, etc.) compared to CPython. Results show compilation significantly improves speed, reduces energy consumption, and optimizes memory, with Codon, PyPy, and Numba achieving over 90% improvements, while Nuitka provides stable memory optimization.

3、[Unravelling t-strings](https://snarky.ca/unravelling-t-strings/)

Core developer Brett Cannon explains the t-string syntax in Python 3.14. It decomposes string templates into tuples of literals and expressions, allowing developers to customize expression processing logic. (Additional: [Detailed explanation of Python 3.14 t-strings](https://pythoncat.top/posts/2025-04-27-t-string))

4、[I don't like NumPy](https://dynomight.net/numpy/)

A rant piece: NumPy is elegant and clear when handling simple arrays, but when working with complex multi-dimensional arrays, its API design is terrible and lacks intuitiveness.

5、[Introducing Typerdrive: Develop API-Connected Typer Apps at Lightspeed](https://blog.dusktreader.dev/2025/05/13/introducing-typerdrive-develop-api-connected-typer-apps-at-lightspeed)

Typerdrive is a library that extends Typer for developing CLI applications that communicate with APIs. By integrating configuration management, caching mechanisms, error handling, HTTP clients, and logging systems, it significantly simplifies the development process and enhances user experience.

6、[The Database Row That Did and Didn't Exist](https://www.mistys-internet.website/blog/blog/2025/05/13/the-database-row-that-did-and-didnt-exist/)

After upgrading to Django 5, the author encountered a strange database issue. The article shares the troubleshooting process, discovering the problem stemmed from discrepancies between Django model definitions and actual database schema, particularly primary key type mismatches causing empty querysets.

7、[Essential Python Monitoring Techniques You Need to Know](https://last9.io/blog/python-performance-monitoring/)

Explores key techniques for Python application performance monitoring, from basic CPU and memory monitoring to advanced distributed tracing. Shows how to collect critical metrics using tools like psutil and Prometheus, and use visualization to help developers quickly identify performance bottlenecks.

8、[Engineer Python projects like a PRO](https://decodingml.substack.com/p/engineer-python-projects-like-a-pro)

Shares best practices for building production-grade Python projects, from project structure to modern toolchain usage. Examples include using uv, ruff, and other tools to improve development efficiency, achieving environment consistency through Docker. Particularly emphasizes experience in modular design, code organization, testing strategies, and deployment processes.

9、[Did Musk steal his new X logo?](https://kenschutte.com/x-glyph/)

The author used Python to analyze the SVG data of X company's new logo, comparing it with X characters from multiple fonts. Demonstrates the powerful capabilities of Python tools (like svg.path, matplotlib, and fontTools) in graphics processing and font analysis.

10、[Moving all our Python code to a monorepo: pytendi](https://attendi.nl/moving-all-our-python-code-to-a-monorepo-pytendi/)

The Attendi team shares challenges and solutions encountered when merging multiple Python projects into a monorepo. They adopted Polylith architecture, dividing code into components, bases, projects, and development sections. Through unified dependency management and simplified code sharing processes, they improved code reusability and development efficiency.

11、[FastAPI Log Tracing: From Theory to Implementation](https://juejin.cn/post/7433830412092801074)

How to implement log tracing in FastAPI projects, especially for distributed services? The article introduces generating unique trace identifiers through middleware, using contextvars to solve log confusion in concurrent requests, and finally combining loguru's log filters to dynamically populate trace information, achieving log tracing in distributed systems.

12、[How Cursor Indexes Codebases Fast](https://read.engineerscodex.com/p/how-cursor-indexes-codebases-fast)

Cursor achieves efficient codebase indexing through Merkle trees. It first chunks code and calculates hash values, builds tree structures, then syncs with servers. This design allows Cursor to quickly identify changed files, uploading only modified parts, significantly reducing bandwidth consumption. It ensures code security through path obfuscation and local storage, making team collaboration more efficient.

## [🐿️ Projects & Resources](https://xiaobot.net/p/python_weekly)

1、[deer-flow: a community-driven Deep Research framework](https://github.com/bytedance/deer-flow)

A community-driven deep research framework that combines language models with specialized tools like web search, content scraping, and Python code execution, while giving back to the community. (10.1K stars)

2、[codeflash: Optimize your code automatically with AI](https://github.com/codeflash-ai/codeflash/tree/main)

An AI code optimization tool designed specifically for Python developers, automatically improving code performance while ensuring functional correctness. Already used by teams from well-known projects like Pydantic, Langflow, and Albumentations.

3、[Scrapling: An undetectable, powerful, flexible, high-performance Python library to make Web Scraping Easy and Effortless](https://github.com/D4Vinci/Scrapling)

A high-performance Python web scraping library supporting multiple selectors (CSS, XPath, regex, etc.), automatic similar element location, and intelligent content extraction features. (5.2K stars)

4、[openpilot: An operating system for robotics](https://github.com/commaai/openpilot)

A driver assistance system supporting over 300 vehicle models, providing complete autonomous driving solutions including lane keeping, adaptive cruise control, and other features. (53.8K stars)

5、[bracket: Selfhosted tournament system](https://github.com/evroon/bracket)

A tournament management system based on FastAPI and Next.js, supporting various formats like single elimination, round-robin, and Swiss tournaments. Features drag-and-drop match scheduling, multi-stage tournament structures, team management, multi-club support, etc. Supports Docker deployment and multiple languages. (1.1K stars)

6、[OpenManus: Open-source Manus AI agent replication](https://github.com/henryalps/OpenManus)

An open-source project dedicated to replicating Manus AI agent capabilities, supporting multi-agent collaboration to autonomously complete complex tasks like travel planning, data analysis, and content generation. Uses modular and containerized design, integrating various tools like web browsing and code execution.

7、[asyncmq: Supercharge your async applications with tasks](https://github.com/dymmond/asyncmq)

Based on asyncio and anyio, supports multiple backends (Redis, Postgres, MongoDB), providing advanced features like delayed tasks, scheduled tasks, error retry, and dead letter queues. Built-in rate limiting and concurrency control, supports task progress reporting and event hooks, seamlessly integrates with ASGI frameworks like FastAPI.

8、[ZeroSearch: Incentivize the Search Capability of LLMs without Searching](https://github.com/Alibaba-NLP/ZeroSearch)

An innovative framework open-sourced by Alibaba DAMO Academy that enhances large models' search capabilities through reinforcement learning without actually calling search engines. Uses simulated search for training, gradually improving model reasoning capabilities through curriculum learning mechanisms.

9、[system_prompts_leaks: Collection of leaked system prompts](https://github.com/asgeirtj/system_prompts_leaks)

Collects leaked system prompts from multiple mainstream large models, including latest versions of ChatGPT, Claude, Gemini, Grok, and other models. Very valuable for researching large models' internal mechanisms and developing related applications. (4.6K stars)

10、[wtfis: Passive hostname, domain and IP lookup tool for non-robots](https://github.com/pirxthepilot/wtfis)

A beautiful command-line tool that integrates multiple OSINT services (like Virustotal, Shodan, Greynoise, etc.) to collect information about domains, FQDNs, and IPs. Supports various data sources including malware detection, geolocation, open ports, and other information. (1.5K stars)

11、[FlexGet: Multipurpose automation tool for content](https://github.com/Flexget/Flexget)

Powerful content automation tool supporting processing of torrents, newsgroups, podcasts, comics, TV shows, movies, and other content types. Obtains content through RSS feeds, HTML pages, CSV files, search engines, and other sources, providing a rich plugin system. (1.8K stars)

12、[TUIFIManager: A cross-platform terminal-based file manager](https://github.com/GiorgosXou/TUIFIManager)

A file manager for terminals, particularly suitable for use in Termux environments. Supports Vim-style shortcuts, provides basic functions like file copying, moving, renaming, as well as advanced features like file search and property viewing. Supports custom themes, shortcuts, and commands. (748 stars)
