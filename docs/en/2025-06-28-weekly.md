# Python Trending Weekly #108：Will AI Replace Junior Developers?

Hey there! I'm Python Cat. Every week I curate the best Python, AI, and tech content for you. This newsletter is open source on [Github](https://github.com/chinesehuazhou/python-weekly), so drop us a star if you dig it! ⭐

This week we're sharing 12 articles, 12 open source projects, 2 podcasts & videos.

## [🦄Articles & Tutorials](https://weekly.pythoncat.top)

1、[My First Open Source AI Generated Library](https://lucumr.pocoo.org/2025/6/21/my-first-ai-library/)

Flask creator Armin Ronacher shares his experience building an XML parsing library with AI. He had Claude write a fault-tolerant XML parser from scratch to handle malformed XML content generated by LLMs. The result? 1,100 lines of parser code, 1,000 lines of tests, complete CI setup, and PyPI publishing—all accomplished in just 30-45 minutes.

2、[Fun with uv and PEP 723](https://www.cottongeeks.com/articles/2025-06-24-fun-with-uv-and-pep-723)

Discover how to create self-contained Python scripts using uv and PEP 723 inline script metadata. By embedding dependency and Python version info directly in your scripts, uv can automatically create virtual environments and install dependencies, making Python scripts as executable as Go programs.

3、[Will AI Replace Junior Developers? I Asked Experts at Pycon US](https://blog.adarshd.dev/posts/pycon-us-ai-and-future-of-programming/)

The author interviewed Python community leaders at PyCon US, including Guido van Rossum and Anthony Sottile. The consensus? AI won't replace developers but will change how we work. They emphasized that understanding code matters more than relying on AI generation, with AI being best suited for boilerplate and repetitive tasks.

4、[Fixing Python Properties](https://willmcgugan.github.io/fixing-python-properties/?featured_on=pythonbytes)

Learn how to use descriptors as an alternative to @property decorators, solving type checking errors when property setters accept flexible types, enabling more natural property value assignment.

5、[Python can run Mojo now](https://koaning.io/posts/giving-mojo-a-spin/)

Explore Python's new ability to call Mojo code. In prime number calculation tests, the Mojo version ran 40x faster than pure Python and 23x faster than NumPy, showcasing Mojo's impressive performance advantages.

6、[Django: Introducing inline-snapshot-django](https://adamj.eu/tech/2025/06/24/django-introducing-inline-snapshot-django/)

The author open-sourced inline-snapshot-django, a package specifically designed for SQL query snapshot testing in Django projects. This tool automatically captures and compares SQL query fingerprints through context managers, providing more detailed debugging info and automated snapshot management compared to Django's native methods.

7、[Cut Django Database Latency by 50-70ms With Native Connection Pooling](https://saurabh-kumar.com/articles/2025/06/cut-django-database-latency-by-50-70ms-with-native-connection-pooling/)

Django 5.1 introduces native database connection pooling, reusing PostgreSQL connections instead of complex new connection creation methods. This reduces database connection overhead by 60-80% and improves response times by 10-30%. The article covers configuration steps, connection pool parameters, and performance monitoring.

8、[The Waiting Time Paradox, or, Why Is My Bus Always Late?](https://jakevdp.github.io/blog/2018/09/13/waiting-time-paradox/)

Using Python simulations and probability theory, the author reveals the waiting time paradox: when buses arrive every 10 minutes on average, passengers also wait 10 minutes on average, not 5. The article explains this phenomenon using Poisson processes and exponential distributions, then validates theoretical predictions with real Seattle bus data—though real-world patterns still differ from theory.

9、[Kalman and Bayesian Filters in Python](https://github.com/rlabbe/Kalman-and-Bayesian-Filters-in-Python/blob/master/README.md)

An interactive tutorial written in Jupyter Notebooks, focusing on the principles and implementation of Kalman and Bayesian filters. These algorithms are widely used in aerospace, robotics, and finance. Starting from sensor noise problems, the author provides accessible explanations through Python code and visualizations.

10、[Switching from virtualenvwrapper to direnv, Starship, and uv](https://treyhunner.com/2024/10/switching-from-virtualenvwrapper-to-direnv-starship-and-uv/)

The author shares their migration experience from virtualenvwrapper to modern Python environment management tools. By configuring direnv for automatic virtual environment activation in project directories, using venv and workon commands to simplify virtual environment creation and project switching, and customizing clean command-line prompts with Starship.

11、[How to think about agent frameworks](https://blog.langchain.com/how-to-think-about-agent-frameworks/)

The LangChain team provides deep analysis of agent framework design philosophy, identifying that the core challenge in building reliable agent systems is ensuring LLMs have appropriate context at each step. The article distinguishes between Workflow (predefined code paths) and Agent (LLM dynamic control) modes, emphasizing that most production agent systems combine both approaches, and compares design philosophies across 100+ agent frameworks.

12、[Lessons From 9 More Years of Tricky Bugs](https://henrikwarne.com/2025/06/15/lessons-from-9-more-years-of-tricky-bugs/)

The author summarizes lessons learned over the past 9 years, analyzing 12 key issues across coding, testing, and debugging dimensions. Topics include null handling, date logic, data format upgrades, and forgetting to push changes. The piece emphasizes the importance of good logging, colleague discussions, and reproducing issues with minimal cases.

## [🐿️Projects & Resources](https://weekly.pythoncat.top)

1、[gemini-cli: An open-source AI agent that brings the power of Gemini directly into your terminal](https://github.com/google-gemini/gemini-cli)

Google's open-source command-line AI workflow tool that can query and edit large codebases with support for over 1 million token context windows. It can generate new applications from PDFs or sketches, automate operational tasks like querying PRs or handling complex code merges, and connect to various tools and media generation capabilities through MCP servers. (33.8K stars)

2、[nxtscape: Nxtscape is an open-source agentic browser](https://github.com/nxtscape/nxtscape)

An open-source AI agent browser with privacy-first design, serving as an alternative to Arc, Dia, and Perplexity Comet. You can use your own API keys or run local models through Ollama. (1K stars)

3、[augmentoolkit: Create Custom LLMs](https://github.com/e-p-armstrong/augmentoolkit)

A data generation tool designed specifically for creating domain expert AI. Upload documents and generate fully trained custom LLMs with one click. Supports offline operation without external API keys, enabling AI to learn domain-specific knowledge and become subject matter experts. (1.6K stars)

4、[BitNet: Official inference framework for 1-bit LLMs](https://github.com/microsoft/BitNet)

Microsoft's open-source official inference framework for 1-bit large models, supporting fast lossless inference for models like BitNet b1.58. Achieves 1.37x to 5.07x speedup on ARM CPUs with 55.4% to 70.0% energy reduction, and 2.37x to 6.17x speedup on x86 CPUs with 71.9% to 82.2% energy reduction. Can run 100B parameter BitNet models on a single CPU at human reading speed. (20.3K stars)

![](https://img.pythoncat.top/2025-06-27-bitnet.jpg)

5、[local-deep-research: AI-powered assistant for deep, iterative research](https://github.com/LearningCircuit/local-deep-research)

An AI-powered research assistant that transforms complex questions into comprehensive cited reports, spanning diverse knowledge sources including academic databases, scientific repositories, web content, and private document collections. Features privacy-first design, flexible configuration, comprehensive analysis, and transparent citations with multiple research modes and advanced features. (3K stars)

6、[pyfuze: Package Python projects into executables](https://github.com/TanixLu/pyfuze)

A tool for packaging Python projects into executables, built on cosmopolitan and uv. Offers three packaging modes: Bundle mode includes Python and all dependencies, Online mode generates compact cross-platform packages, and Portable mode creates standalone cross-platform executables. Supports macOS, Linux, and Windows platforms.

7、[nbdev: Create delightful software with Jupyter Notebooks](https://github.com/AnswerDotAI/nbdev)

Write notebooks using lightweight markup and automatically generate high-quality documentation, tests, continuous integration, and packaging. Supports automatic documentation generation hosted on GitHub Pages, publishing to PyPI and conda, bidirectional sync between notebooks and source code, and Git-friendly notebook processing. (5.1K stars)

8、[fastapi_mcp: Expose your FastAPI endpoints as Model Context Protocol (MCP) tools, with Auth!](https://github.com/tadata-org/fastapi_mcp)

A library for exposing FastAPI endpoints as MCP tools with built-in authentication. Supports zero or minimal configuration, flexible deployment, direct communication with FastAPI apps using ASGI transport, and provides native dependency injection with unified infrastructure. (5.6K stars)

9、[Awesome-ML-SYS-Tutorial: My learning notes/codes for ML SYS](https://github.com/zhaochenyang20/Awesome-ML-SYS-Tutorial)

A collection of machine learning notes and code covering core topics like RLHF system development, SGLang learning, scheduling and routing, and distributed training. Includes in-depth source code analysis, practical experience sharing, and development guides—a quality resource for learning ML + SYS. (2.7K stars)

10、[machine-learning-visualized: ML algorithms implemented and derived from first-principles in Jupyter Notebooks and NumPy](https://github.com/gavinkhung/machine-learning-visualized)

Visual tutorials implementing and deriving machine learning algorithms from first principles using Jupyter Notebooks. Each algorithm includes complete mathematical derivations and training process visualizations, covering neural networks, logistic regression, perceptrons, PCA, K-means clustering, and gradient descent. Also provides Marimo-based interactive notebooks to observe how weights affect loss functions.

![](https://img.pythoncat.top/2025-06-27_neural_network.png)

11、[code-graph-rag: Search Monorepos and get relevant answers](https://github.com/vitali87/code-graph-rag)

A graph-based multi-language codebase retrieval-augmented generation system that uses Tree-sitter to parse code and build knowledge graphs. Enables natural language queries of codebase structure and relationships.

12、[FeedMe: AI-powered RSS reader, deployable to GitHub Pages or with Docker](https://github.com/Seanium/FeedMe)

A lightweight RSS reader that redefines the RSS reading experience with AI, supporting multi-source RSS aggregation, AI summary generation, scheduled update mechanisms, categorized browsing, and theme switching. Easily deployable to GitHub Pages or locally with Docker.
