## [🦄Articles & Tutorials](https://weekly.pythoncat.top)

1、[330× faster: Four different ways to speed up your code](https://pythonspeed.com/articles/different-ways-speed/)

Using the example of counting letter frequencies in English novels, this article demonstrates four different code optimization approaches: achieving 2x improvement by reducing redundant calculations, 10x speedup using Rust implementation, further performance gains through parallel multi-core processing, and process optimization to ensure measurement accuracy, ultimately achieving a 330x performance boost.

2、[Why This Python Performance Trick Doesn't Matter Anymore](https://blog.codingconfessions.com/p/old-python-performance-trick)

Creating function local aliases in Python is a traditional trick for optimizing loop performance. This article analyzes bytecode and virtual machine implementation mechanisms to reveal how CPython 3.11's instruction specialization optimization makes this manual optimization unnecessary.

3、[Python 3.14's Further Performance Evolution: Tail Call Interpreter](https://www.manjusaka.blog/posts/2025/07/02/tail-call-in-3-14-interpreter/)

Python 3.14 introduces the Tail Call Interpreter mechanism, which splits massive switch-case structures into multiple small functions and leverages tail call optimization to solve compatibility issues with traditional Computed Goto across different compiler versions, improving overall performance predictability.

4、[Uncommon Uses of Python in Commonly Used Libraries](https://eugeneyan.com/writing/uncommon-python/)

The author discovered 6 uncommon but practical programming techniques in the source code of well-known Python libraries: using super() in base classes for cooperative multiple inheritance, implementing optional features with Mixins, using relative imports to avoid naming conflicts, simplifying API interfaces in __init__.py, and leveraging empty conftest.py files to automatically configure pytest paths.

5、[Code reading: The python std lib module - shelve.py](https://www.beyonddream.me/post-5/)

A deep dive into the source code implementation of Python's standard library shelve module, demonstrating how to create persistent dictionary objects. Internally, it serializes arbitrary Python objects through pickle and delegates storage to the dbm module. The article provides detailed analysis of caching mechanisms, context manager design, and other elegant programming techniques.

6、[How we built our multi-agent research system](https://www.anthropic.com/engineering/built-multi-agent-research-system)

Anthropic developed a multi-agent research system where a lead agent coordinates multiple sub-agents to search information in parallel, achieving a 90.2% performance improvement over single-agent systems. The article shares valuable insights on system architecture design, prompt engineering techniques, and production environment reliability challenges.

7、[Programming Language Design in the Era of LLMs: A Return to Mediocrity?](https://kirancodes.me/posts/log-lang-design-llms.html)

The author explores LLM's impact on Domain-Specific Language (DSL) design, noting that while LLMs excel with mainstream languages like Python, their performance drops dramatically with niche languages, effectively doubling the opportunity cost of DSLs. The article proposes three response strategies: teaching LLMs to understand DSLs through Python as an intermediate language, designing new DSLs that can collaborate with LLMs, and developing specification languages for validating LLM-generated code.

8、[How global variables work in Python bytecode](https://fromscratchcode.com/blog/how-global-variables-work-in-python-bytecode/)

An in-depth analysis of Python global variables' bytecode implementation mechanism, revealing how the virtual machine handles global variables through dynamic name resolution while local variables use index access. The article analyzes global storage principles through the LOAD_GLOBAL instruction, explaining how this design enables Python's dynamic features, including runtime global variable modification and cross-module monkey patching.

9、[Comprehensive Analysis of Python Regular Expressions](https://www.cnblogs.com/kuangdaoyizhimei/p/18925535)

A comprehensive introduction to Python regular expressions usage and principles, covering basic metacharacters, escape characters, extensive examples (such as greedy matching, non-greedy matching, zero-width assertions), and advanced regex features (such as capture groups, non-capture groups, named groups) and other advanced characteristics.

10、[JavaScript Promise Integration in Pyodide](https://blog.pyodide.org/posts/jspi/)

Pyodide solves the compatibility issue between synchronous Python code and asynchronous JavaScript APIs in browsers through JSPI technology, introducing the run_sync() function to implement stack switching mechanisms, allowing synchronous Python code to call asynchronous JavaScript without modifying the entire call stack.

11、[Writing Toy Software Is A Joy](https://blog.jsbarretto.com/post/software-is-joy)

The author advocates writing toy programs to rediscover programming joy, following the 80:20 principle to implement core functionality with minimal code. The article shares 20 suitable practice toy projects, including regex engines, operating system kernels, game emulators, and compilers, with each project annotated with difficulty and time estimates.

12、[My Thoughts on Various AI Coding Agent Tools](https://xxchan.me/ai/2025/06/08/ai-coding.html)

Analyzes the advantages, disadvantages, and development trends of current mainstream AI programming tools (Cursor, GitHub Copilot, Claude Code, Amp, OpenAI Codex, Devin, v0, etc.).

13、[Reflections on Vibe Coding After Some Time](https://limboy.me/posts/vibe-coding)

Compares different programming assistant tools (such as Cursor and Claude Code) in terms of code understanding and generation quality experience, summarizing practical Vibe Coding techniques such as precise goal decomposition, providing key inputs, timely intervention, and making good use of planning mode. (Additional: Another AI-assisted programming experience sharing [How I Vibe Coding?](https://xuanwo.io/2025/03-how-i-vibe-coding/))
